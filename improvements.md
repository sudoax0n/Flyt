# 📈 Flyt Improvements & Enhancements Log (improvements.md)

This log tracks the current architecture, planned enhancements, and recommendations for **Flyt** (formerly DrosUI). Agents should read and update this log as they develop.

---

## 🏛️ Current System Architecture
- **Backend Tracker (`tracker.py`)**: Uses CPU-only OpenCV background subtraction (`MOG2`) and contour size filtering to track two flies in a single arena. Calculates speed, distance (proximity), and overall activity.
- **Orchestration Server (`server.js`)**: An Express server handling file uploads, spawning the Python process asynchronously, and providing job status polling.
- **React Frontend (`App.jsx`)**: Uses Recharts to visualize velocity, inter-fly proximity, and a spatial density heatmap.

## 🎥 Target Video Assay Specifications
The primary focus of Flyt is a **top-down view of a square acrylic mating chamber** containing two flies (typically a male and a female).

### Physical Environment Constraints & Optimizations:
1.  **High-Texture Mesh Bottom**: The background of the chamber features a high-contrast grid mesh. 
    *   *CV Optimization*: Utilize dynamic `MOG2` parameters to model background pixel variance. Enforce morphological closing (`5x5` kernel) to prevent grid lines from breaking fly contours.
2.  **Corner Cotton Plug (High Contrast Static Blob)**: A white cotton plug is located in the corner of the chamber.
    *   *CV Optimization*: Set a strict upper-bound contour area filter in `tracker.py` (e.g., `30 < area < 250` pixels) to automatically ignore any massive contours triggered by cotton movement or fiber adjustments.
3.  **Reflective Acrylic Walls**: The plastic borders of the chamber can create mirror reflections of flies.
    *   *CV Optimization*: Provide an interactive Region of Interest (ROI) cropping utility in the UI to truncate the plastic borders and restrict tracking to the inner mesh field.
4.  **Size Dimorphism (Male vs. Female)**: In this viewport, the female fly has a larger, wider footprint while the male is smaller and more elongated.
    *   *CV Optimization*: Leverage this sizing delta inside the re-identification algorithm (**ToxId-Light**) to resolve post-collision identities.

### 1. 🎥 H.264 Video Transcoding (High Priority)
- **Current Issue**: The dashboard embeds a video player targeting `/tracked.mp4` which is simply a copied version of the *original raw upload*. The actual tracked video with bounding boxes and lines (`tracked_raw.mp4` generated by Python using `mp4v` codec) is not HTML5-compatible, so the visual tracking feedback is bypassed.
- **Solution**: 
  - Install `ffmpeg-static` in the Node backend.
  - After `tracker.py` finishes with code `0`, run an asynchronous `ffmpeg` child process to transcode `tracked_raw.mp4` to H.264 `tracked.mp4` (e.g., `ffmpeg -i tracked_raw.mp4 -vcodec libx264 -acodec aac -y tracked.mp4`).
  - Delete `tracked_raw.mp4` once completed.

### 2. 🔀 Overlap / Crossing Resolution (High Priority)
- **Current Issue**: When two flies touch or cross paths, they merge into one contour. The current code assigns both fly coordinates to the merged center and sets proximity to 0. When they separate, it is highly prone to **identity swapping** because of simple nearest-neighbor Euclidean distance calculations.
- **Solution (ToxId-Light & Hungarian Matching)**:
  - **Bipartite Assignment**: Replace the manual coordinates swap code in `tracker.py` with a proper bipartite linear assignment solver (e.g., using `scipy.optimize.linear_sum_assignment` or a lightweight pure Python Kuhn-Munkres implementation).
  - **Feature Signatures (ToxId-Light)**: During frames where 2 contours are detected, maintain a running history of:
    - **Contour Area size** (male flies are typically smaller than females).
    - **Hu-Moments** (`cv2.HuMoments(cv2.moments(contour))`), which provide a rotation, scale, and translation invariant geometric fingerprint of each fly's shape.
  - **Overlap State**: When only 1 merged contour is detected, freeze tracking updates and mark the overlap state.
  - **Resolution**: Upon separation back to 2 contours, calculate the Euclidean matching cost on the bipartite graph between current blobs and historical signatures (combining area delta and Hu-moments distance). Solve the assignment matrix to reliably restore fly track identities.

### 3. ⚙️ Connecting Dashboard Settings (Medium Priority)
- **Current Issue**: The Settings Modal in React allows adjusting parameters (sensitivity, arena type, FPS), but they are local state variables and are never sent to the backend.
- **Solution**:
  - Update `POST /api/upload` to accept configuration parameters (either as query params or multipart fields).
  - Update `server.js` to parse these configurations and pass them as CLI arguments (e.g., `--sensitivity`, `--arena`, `--fps`) when spawning `tracker.py`.
  - Update `tracker.py` to parse these arguments and adjust the MOG2 background subtractor and metrics calculations accordingly.

### 4. 📊 PDF, GraphPad Prism, and Universal Visualizer (Medium Priority)
- **Current Issue**: The "Export PDF" and "Download CSV (Prism)" buttons on the dashboard are visual placeholders. Additionally, heavy ML tools (SLEAP, DeepLabCut) lack clean user-friendly visualization dashboards.
- **Solution**:
  - **Universal Visualizer Import**: Add an upload parser in the React dashboard (`App.jsx`) to accept standardized Tidy CSV exports from DeepLabCut and SLEAP. This allows researchers to utilize offline deep learning pose models and load their coordinates directly into Flyt's premium web UI for interactive velocity, sleep, and spatial heatmaps.
  - **Prism Formatting**: Implement a CSV converter to pivot raw tracking frame rows into column-oriented tables (e.g., `Time`, `Fly1_Velocity`, `Fly2_Velocity`, `Distance`) ready for GraphPad Prism.
  - **Report Generation**: Integrate client-side PDF export (using `jspdf` and `html2canvas`) to compile overview charts and metrics cards into a clean, publishable lab report.

### 7. 🎯 Behavior Selection & Feasibility Scorecard (Medium Priority)
- **Current Issue**: Different assays require different metrics, and varying video quality (glare, contrast, resolution) can degrade tracking accuracy silently.
- **Solution**:
  - **UI Selection**: Add checkable cards in the video upload screen (`UploadView`) allowing researchers to toggle which behaviors they wish to quantify (Locomotion, Sleep, Courtship Index).
  - **Diagnostic Pass**: Execute a fast, preliminary analysis pass on the first frame of the uploaded video to calculate average pixel contrast, average fly blob size (area in pixels), and frame noise.
  - **Scorecard Display**: Show a diagnostic scorecard rating expected accuracy for each selected behavior (e.g., *Locomotion: 98% (Excellent Contrast)*, *Wing Extension: 40% (Warning: Low resolution)*) to warn researchers of quality issues before running a 10-hour track.

### 8. 📌 Human-in-the-Loop Timestamp Verification (High Priority) — **DONE (2026-06-15)**
- **Shipped**: `tracker.py` writes `public/events.json` (`courtship_bout` + `low_confidence_segment`). `server.js` serves `/api/events` and `/api/verification`; resets verdicts on new upload. Dashboard verification panel: seek video, confirm/reject, **verified / detected** bout stat.
- **Note**: Wide-FOV test videos (e.g. `Drosophila_courtship_and_copulation_1080P.mp4`) may yield **0 events** at default thresholds — use `fly_video.mp4` or a top-down chamber clip for bout QC. Sleep metric still client-side (K-05 partial).

### 9. 🗃️ Automatic Multi-Vial Arena Segmentation & Cropping (High Priority - Scalability)
- **Current Issue**: High-throughput assays record multiple vertical tubes or vials (e.g., a 1x12 or 3x4 grid) in a single wide-angle frame. Manually cropping these into 12 separate video files takes researchers hours of manual editing.
- **Solution**:
  - **Vial Detection Algorithm**: Extract a single diagnostic frame. Apply adaptive thresholding and morphological operations to identify high-contrast vertical boundaries. Run contour analysis to filter for bounding boxes matching the aspect ratio of standard vials.
  - **UI Confirmation Grid**: Display the diagnostic frame with the 12 detected bounding boxes overlaid on the dashboard, prompting the user to approve or adjust the grid.
  - **Concurrent Ffmpeg Cropping**: Spawn 12 parallel non-blocking `ffmpeg` crop processes (e.g., `ffmpeg -i input.mp4 -filter:v "crop=W:H:X:Y" -c:a copy vial_01.mp4`) in Node.js to cut the main video into 12 individual files in seconds.
### 10. 📏 Interactive Scale Calibration (Pixels → Millimeters) (Medium Priority)
- **Current Issue**: Speed, distance, and proximity metrics are calculated in pixels, but scientific publications require metric units (mm or cm).
- **Solution**:
  - **Calibration Tool**: Implement an interactive line-drawing tool on the first frame of the uploaded video in the dashboard UI. Let the researcher input the physical length of the line (e.g. 15 for a 15mm chamber width).
  - **Scaling Matrix**: Calculate the pixel-to-mm ratio ($\text{Ratio} = \text{physical\_length} / \text{pixel\_length}$). Pass this ratio as a parameter to the tracking engine to multiply all trajectory coordinates, velocities, and inter-fly distances, exporting native, publication-ready metric tables.

### 11. 🎯 Multi-Arena ROI Masking (Circle & Square Drawing) (Medium Priority)
- **Current Issue**: Background noises (cotton plug fibers, reflections along plastic chamber walls, paper labels) can trigger false tracking contours.
- **Solution**:
  - **Visual ROI Selection**: Present the first video frame in the dashboard and allow users to drag and drop custom circular masks (for petri dishes) or rectangular boxes (for mating chambers).
  - **Binary Mask Generation**: Generate a binary ROI mask on the server side and pass it to OpenCV. The tracker zeroes out all pixels outside the defined ROI shapes, isolating tracking exclusively to the interior mesh field of the chambers.

## 🔮 Suggested Future Scope

### 🕸️ Social Spacing & Network Analysis (Group Assays)
- **Concept**: Expand Flyt to support group behavior phenotyping (e.g. 8–10 flies in a single large petri dish) by tracking multiple coordinate trajectories concurrently.
- **Metrics**: 
  - Calculate **Nearest Neighbor Distance (NND)** and spatial dispersion over time.
  - Compile a dynamic **Social Interaction Network Graph** (using Node-link libraries) to visualize which individuals spent the most time interacting, indicating social structure or territorial dominance.

- **Concept**: Integrate Google's lightweight open-weights LLMs/VLMs locally on the lab's machine to assist researchers with natural language analytics, results drafting, and behavioral verification.
- **Features**:
  - **Ethology Co-Pilot (Gemma 2 2B)**: Inject the final behavioral metrics JSON directly into a local Gemma prompt. The researcher can chat with their dataset: *"Is a Courtship Index of 0.38 normal for wild-type flies?"*, *"Draft the Results section of my paper based on this run"*, or *"Generate R code to run a t-test against control group speed."*

  - **Data Privacy**: Running models locally ensures unpublished biological datasets remain private and never leave the lab server.
